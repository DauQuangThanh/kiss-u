# Roadmap styles

Pick one style and stick to it. Mixing styles confuses stakeholders.

## Now / Next / Later (preferred for uncertain teams)

- **Now** — in flight; dates not exact, but close.
- **Next** — starts after Now finishes; could be named but not
  committed.
- **Later** — direction, not promise; can change.

Good when: early-stage product, high uncertainty, agile cadence.

## Date-based (good for contracted / waterfall work)

- **Quarter 1** — end dates named. Often paired with a release plan.
- **Quarter 2** — …

Good when: compliance deadlines, contract milestones, hardware
ship dates.

## Hybrid

Named date for first window; Now/Next/Later for the rest. Works
when only the first slice is firm.

## AI-authoring note

Roadmaps are a **communication artefact** first. Write the rationale
for each window in plain business language; technical detail belongs
in the plan, not the roadmap.

# Sprint goal — guide

A sprint goal is a **one-sentence** statement of the outcome the
team commits to. It is *not* a task list. It answers: "what will
be true about the product at the end of this sprint that is not
true today?".

## Good examples

- "Customers can sign in with Google SSO end-to-end on the staging
  environment."
- "Operators receive a weekly usage-summary email, including churn
  signals."
- "The checkout API returns <200 ms p95 under 500 RPS."

## Bad examples

- "Work on SSO." — too vague; no outcome.
- "Finish stories BL-14, BL-15, BL-22." — a task list, not a goal.
- "Improve performance." — unmeasurable.

## Link goal → items

Every item committed to the sprint should be traceable to the
goal. If an item doesn't obviously support the goal, it's either
a bug-fix slot or scope creep — either way, flag it.

## AI-authoring note

When the user hasn't stated a goal, propose 2–3 candidate goals
derived from the top backlog items and ask them to pick. Never
invent a goal without surfacing the proposal.

# Wireframe fidelity — what this skill does and doesn't do

| Level | Produced by | What it is |
|---|---|---|
| **Lo-fi (text / ASCII)** | ✅ this skill | Structure, content, layout intent |
| **Wireframe (shapes in Figma)** | ❌ human designer | Visual structure, components named |
| **Mockup (styled)** | ❌ human designer | Typography, colour, brand |
| **Prototype (clickable)** | ❌ human designer | Interaction + transitions |

## What counts as lo-fi

- Boxed regions for content.
- Labels for every interactive element.
- One state sketched per screen; additional states listed.
- No pixel dimensions, no colours, no typography.

## When to escalate

- If the user needs to decide between two visual directions, refer
  them to a human designer or a design system. The AI cannot make
  that call.
- If the flow needs validation with real users, flag that
  separately — **this skill does not conduct research**.

## User flows

Mermaid flowcharts only. Every node is a screen or a decision.
Edges are transitions labelled with the user action.

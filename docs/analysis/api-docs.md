### 1.3 `kiss init [PROJECT_NAME]`

- Source: `cli/init.py:76-87`.
- Description: "Initialize a new kiss project. Scaffolds the project
  entirely from assets bundled inside the `kiss` package ‚Äî no network
  access is required or performed." (`cli/init.py:88-108`)
- Positional argument:
  - `project_name: str` ‚Äî name of the new directory (`cli/init.py:78`).
    Use `.` to mean current directory (`cli/init.py:125-127`).
- Flags:
  - `--ignore-agent-tools` ‚Äî skip CLI tool detection (`cli/init.py:79`).
  - `--no-git` ‚Äî skip git init (`cli/init.py:80`).
  - `--here` ‚Äî initialise in current directory (`cli/init.py:81`).
  - `--force` ‚Äî merge into non-empty directory (`cli/init.py:82`).
  - `--preset <id>` ‚Äî install a preset during init (`cli/init.py:83`).
  - `--branch-numbering <sequential|timestamp>` (`cli/init.py:84`).
  - `--integration <key>` ‚Äî non-interactive single-integration install
    (`cli/init.py:85`).
  - `--integration-options "<flags>"` ‚Äî passed through to the
    integration's option parser (`cli/init.py:86`,
    `kiss_cli/__init__.py:103-148`).
- Exit codes (caught by `raise typer.Exit(N)`):
  - `0` ‚Äî success (default).
  - `0` ‚Äî user cancelled the merge prompt (`cli/init.py:158`).
  - `1` ‚Äî bad combinations / unknown integration / missing required
    options / directory conflict (multiple sites; representative:
    `cli/init.py:122,131,135,140,165,183,188,205,213`,
    `cli/init.py:494-508`).
- Side effects:
  - Creates the project directory if missing.
  - Writes `.kiss/context.yml`, `.kiss/integration.json`,
    `.kiss/init-options.json`, per-integration manifests, agent skill
    folders for every selected AI provider, role-agent subagents,
    optional bundled `git` extension, optional bundled `kiss`
    workflow, optional preset.
  - Optionally `git init` + first commit.
  - On failure with `--here=False` and a freshly-created directory,
    `shutil.rmtree(project_path)` cleans up (`cli/init.py:506-507`).

## 3. Build-hook contract (Hatch)

KISS plugs into Hatchling via a custom build hook:

- Class: `CustomBuildHook(BuildHookInterface)` ‚Äî
  `scripts/hatch_build_hooks.py:104`.
- Hook method: `initialize(self, version, build_data)` ‚Äî
  `scripts/hatch_build_hooks.py:107`. Hatchling calls this *before*
  collecting wheel files.
- Wired by `pyproject.toml:29-30`:

  ```toml
  [tool.hatch.build.hooks.custom]
  path = "scripts/hatch_build_hooks.py"
  ```

- Behaviour:
  1. Wipes `build/core_pack/` and re-creates it
     (`hatch_build_hooks.py:113-115`).
  2. Mirrors the `ASSET_MAP` entries
     (`hatch_build_hooks.py:32-48`):
     - `agent-skills/` ‚Üí `agent-skills/`
     - `subagents/` ‚Üí `subagents/`
     - `presets/` ‚Üí `presets/`
     - `extensions/` ‚Üí `extensions/`
     - `workflows/` ‚Üí `workflows/`
     - `integrations/catalog.json` ‚Üí `integrations/catalog.json`
  3. Excludes `.DS_Store`, `__pycache__`, `.pytest_cache`, `*.pyc`,
     `*.pyo`, and any path part starting with `_`
     (`hatch_build_hooks.py:51-61`). The `_` rule keeps developer
     scaffolding such as `agent-skills/_template/` out of the wheel.
  4. Shells out to `python scripts/generate-checksums.py` ‚Äî
     `hatch_build_hooks.py:129-138`. This produces
     `build/core_pack/sha256sums.txt` (`generate-checksums.py:31-68`).
  5. Errors raise `RuntimeError("Asset checksum generation failed")` ‚Äî
     `hatch_build_hooks.py:144`.

- Resulting wheel layout ‚Äî `pyproject.toml:41-42`:

  ```toml
  [tool.hatch.build.targets.wheel.force-include]
  "build/core_pack" = "kiss_cli/core_pack"
  ```

  i.e. `build/core_pack/` becomes `kiss_cli/core_pack/` inside the wheel.

- Sdist layout ‚Äî `pyproject.toml:44-70`. Includes the repo-root asset
  directories so building a wheel from the sdist reproduces the same
  bundle. Excludes `build/`, `__pycache__`, `.pytest_cache`,
  `.DS_Store`, and `*.pyc`.

## 5. Exit-code conventions

- Typer exits with the integer passed to `typer.Exit(...)`.
- `0` ‚Äî success.
- `1` ‚Äî most user-error and validation failures (representative
  references at `cli/init.py:122,131,135,165,183,188,205`,
  `cli/integration.py:217,256,289,317,346,396,420,466,492,505`,
  `cli/workflow.py:108,116,119`).
- `130` ‚Äî keyboard-interrupt during streamed integration dispatch
  (`integrations/base.py:202-207`).
- `kiss check` aggregate run exits with the maximum of the three
  sub-checks (`cli/check.py:613`).
